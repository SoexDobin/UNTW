# 1주차 빅데이터 개론

## 빅데이터 개념

**빅데이터란 기존 데이터 처리 방법으로는 처리하기 어려운 대규모의 다양한 유형의 데이터를 의미합니다.**

## 빅데이터 처리 순환 과정 시험!!
1. 데이터 수집
2. 데이터 저장
3. 데이터 처리
4. 데이터 분석 (해석(시각화) 경과제시/모형수립)
5. 데이터 시각화

## 빅데이터 유형
* 데이터 : 대형 복잡 고속 다양
1. 구조화된 데이터 : 정형화가 되어있고 관계형 데이터베이스나 엑셀과 같이 일정 형식을 따저가며 처리하는 것들을 말한다.
2. 구조화되지 않은 데이터 : 텍스트, 이미지, 오디오, 비디오 등 여러 형태이며 한 형태로써 분석되기보단 각각의 특성에 맞는 음성 분석, 영상 분석 등 크기나 형태가 일정하지 않은 데이터들을 말합니다.
3. 반정형 데이터 : 일부는 구조화 되어있고 일부는 구현되있지 않은 특수 목적의 데이터이며 위 두 데이터를 함께 처리할때 JSON,XML 같이 중관과정을 같이 처리해주기도 한다.

## 빅데이터의 특징
1. 규모(Volume) : 대량의 데이터를 다루기에 기술에 발전에따라 현재는 제타바이트까지 대량의 데이터를 가집니다.
2. 속도(Velocity) : 많은 양의 데이터가 생성되는 만큼 모았다가 일괄 처리가 아닌 실시간 처리 방식을 이용해야 합니다. 인터넷 속도도 그 일부이다.
3. 다양성(Variety) : 정형화 데이터, 비정형화 데이터, 반정형 데이터등 종류가 다양히 존재한다. 멀티미디어의 소셜, 위치, 소비 등 계속 증가하고 있다.
4. 정확성 : 많은 양의 데이터 속에서 알맞은 최고의 데이터를 잡아내햐 하며 많은 양의 데이터를 수집하고 처리하기 때문에 기존의 통계적 분석 방법보다 더욱 정확한 결과를 도출할 수 있습니다.
5. 가치 : 빅데이터 분석은 기업이나 조직이 보유한 데이터를 활용하여 다양한 분야에서 새로운 가치를 창출할 수 있습니다.

## 빅데이터의 이슈화
1. 스마트폰의 보급화로 인한 비정형화 데이터 수집률이 더 증가하였다.
2. 클라우드 서비스의 등장으로 개인과 조직의 데이터가 한 곳에 저장되고 이를 바탕으로 분석 및 활용할 수 있게 되었다.
3. 소셜 미디어의 보급화로 상호 작용 데이터가 증가함에 따라 비즈니스 측면에서 사용하려는 움직임이 많아지게 되었다. 

## 비즈니스 모델의 개념
* 기업으로 하여금 수익을 유지하는 일련의 활동.
* **수익 모델로 써 정의한다.**
* 구조는 크게 데이터 수집, 저장, 분석 등을 담당하는 서버 및 하드웨어와 관리, 분석 등을 실행하는 소프트웨어로 나뉜다.

## 빅데이터의 사회 * 경제적 가치
* 산업의 투명성 증대
* 소비자 니즈 발견 / 트렌트 예측
* 소비자 니즈를 위한 고객 세분화
* 자동 알고리즘을 통한 의사결정과 대
* 비즈니스 모델, 상품, 서비스 혁신

## 빅데이터 기반 서비스 비즈니스 모델
* 이상 현상 감지 시나리오
* 가까운 미래 예측 시나리오
* 현 상황 분석 시나리오

## 빅데이터의 수용 4단계

## 인공지능 알고리즘

1. 지도학습 알고리즘: 데이터를 통해 모델을 학습하고, 새로운 데이터가 들어왔을 때 결과를 예측하는 알고리즘입니다. 대표적으로 회귀, 분류, 신경망 등이 있습니다.

2. 비지도학습 알고리즘: 데이터에 대한 사전 지식 없이, 데이터의 구조를 파악하거나 군집화를 수행하는 알고리즘입니다. 대표적으로 클러스터링, 차원 축소, 자기조직화맵(SOM) 등이 있습니다.

3. 강화학습 알고리즘: 에이전트가 환경과 상호작용하며 보상을 최대화하는 정책을 학습하는 알고리즘입니다. 대표적으로 Q-러닝, SARSA 등이 있습니다.


## 빅데이터의 핵심적 키워드는 분석!

2. 예측 기법 : 과거 데이터와 변수 간의 관계를 이용하여 관심이 되는 변수 추정
3. 데이터 마이닝 기법 : 많은 데이터 속에 숨겨진 유용한 패턴을 추출하여 여러 목적으로 활용
4. 최적화 기법 : 주어진 제한 조건을 만족하며 목적함수를 최소 최대화 하기위한 방법
   
**상황에 맞는 분석으로 패턴을 만들어 근본적 문제를 찾아나서야 함**   
* 연역적 추리 : 주어진 전제로 측정 결론을 이끌어냄
* 귀납적 추리 : 개개의 특수한 사실로 일반적 결론을 이끌어냄

### 데이터 분석과제
* 과제란 해결해야 할 문제나 이슈
* 분석 과제는 분석 활동의 대상이 됨
* 문제는 기대 상태에 현재 상태의 차이라 정의
* 문제가 해결된 상태는 문제가 '0'이 된 상태
* 데이터 분석은 기대 상태를 낮추고 현재 상태를 높임
* 기대, 현재 상태를 동시 조절 하며 원인을 찾고 여러 방안 수렴

### 수요 기반 분석과제
